# 公众号写作工作流

## 工作流总览

建议把公众号写作设计成一条稳定流水线：

```text
选题池 -> 资料收集 -> 观点提炼 -> 大纲 -> 初稿 -> 修改润色 -> 标题封面 -> 排版发布 -> 数据复盘
```

## 1. 选题池

先建立长期选题库，不要每次临时想题。

建议字段：

```text
选题标题
目标读者
核心问题
文章类型
素材来源
优先级
状态
发布时间
复盘数据
```

文章类型可以先分为：

```text
经验总结
教程指南
观点评论
案例拆解
资源清单
```

## 2. 单篇文章生产流程

每篇文章固定走 8 步：

1. 明确读者：这篇写给谁？
2. 明确问题：读者看完能解决什么？
3. 收集素材：案例、数据、经历、引用、截图。
4. 提炼观点：一句话说清文章主张。
5. 写大纲：标题、开头、3-5 个小节、结尾。
6. 写初稿：先完整写出来，不边写边改。
7. 修改润色：删废话、调结构、增强例子。
8. 发布复盘：记录阅读量、点赞、收藏、评论、转发。

## 4. AI 辅助节点

不要只用 AI 来“帮我写一篇文章”，更适合嵌入每个环节：

```text
选题：帮我基于目标读者生成 20 个选题。
大纲：帮我把这个观点拆成公众号文章结构。
初稿：根据大纲写第一版。
改稿：删掉空话，让表达更具体。
标题：生成 20 个标题，按点击欲望和准确性排序。
审稿：检查逻辑跳跃、废话、标题党、风险表述。
复盘：根据数据总结下次优化方向。
```

## 6. 最小可执行版本

第一阶段先只维护 4 个文件：

```text
wechat/topic-pool.md
wechat/public-account-workflow.md
wechat/templates/article-template.md
wechat/reviews/monthly-review.md
```

最开始的目标是：每周稳定产出 1 篇，连续 4 周，并且每篇都做复盘。等流程跑顺后，再增加自动化、素材库、标题库、发布日历。
