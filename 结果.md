# Codex 80% 功能快速学习计划总结

## 学习目标

用 7 天掌握 Codex 的高频能力：读代码、改代码、跑测试、调 bug、做 review、写文档、验证前端、管理权限和长任务。

## 7 天执行计划

| 天数 | 重点 | 练习任务 | 完成标准 |
| --- | --- | --- | --- |
| 第 1 天 | 基本交互、权限、工作区 | 让 Codex 解释项目结构、启动项目、跑测试 | 能让 Codex 找文件、读代码、运行命令 |
| 第 2 天 | 代码阅读 | 解释模块、调用链、入口文件、数据流 | 能快速理解陌生仓库 |
| 第 3 天 | 小功能开发 | 添加一个真实小功能 | 功能完成，能看 diff，能跑测试 |
| 第 4 天 | Debug | 复现、定位、修复一个 bug，并补测试 | 能写清复现步骤和期望行为 |
| 第 5 天 | Code review | 让 Codex 审查 diff 或 PR | 能得到按严重度排序、有行号的 review |
| 第 6 天 | 前端/文档/数据工作流 | 启动前端检查 UI，或生成 README/脚本 | 能完成代码之外的交付任务 |
| 第 7 天 | 综合项目 | 从 issue 到实现、测试、总结 | 能独立交给 Codex 一个小任务并完成 |

## 每天固定流程

1. 给 Codex 明确任务：目标、限制、验收标准。
2. 要求它先读代码，再修改。
3. 每次修改后查看 diff。
4. 让它运行测试、lint 或启动服务验证。
5. 最后要求总结：改了什么、如何验证、还有什么风险。

## 推荐提示词模板

```text
请先阅读相关代码，不要急着修改。
目标：……
约束：不要改无关文件；沿用现有风格；不要引入新依赖，除非必要。
验收标准：
1. ……
2. ……
请完成实现、运行相关测试，并总结改动和风险。
```

## 必须掌握的 10 个高频用法

1. 解释代码库结构。
2. 定位功能所在文件和逻辑。
3. 实现小功能。
4. 复现并修复 bug。
5. 补充测试。
6. 审查代码 diff 或 PR。
7. 在不改变行为的前提下重构。
8. 更新 README、接口说明或迁移文档。
9. 启动本地服务并验证前端。
10. 维护项目规则文件，例如 `AGENTS.md`。

## 达标标准

掌握下面 4 件事，就基本学会 Codex 80% 高频功能：

1. 30 分钟内让 Codex 帮你讲清一个陌生项目的结构。
2. 让 Codex 完成一个小需求，并通过测试。
3. 让 Codex 复现、定位、修复一个 bug，并补测试。
4. 让 Codex 对一个 diff 做出有价值的 code review。

## 核心建议

不要只看教程。直接用真实项目练习：给清楚目标，让 Codex 修改，你审查 diff，再让它验证结果。
